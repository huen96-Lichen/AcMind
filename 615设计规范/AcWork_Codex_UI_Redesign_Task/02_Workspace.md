# 02 工作台

## 页面定位

工作台不是功能入口集合，也不是系统监控页面。

它只回答：

1. 当前正在做什么
2. 接下来应该做什么
3. 哪些内容等待处理
4. AcWork 是否正常运行

## 页面结构

```text
Toolbar
MainContent
├── FocusHeader
├── SummaryRow
├── ActivityColumn
├── TodayColumn
└── SystemStatusStrip
```

## 第一层：当前工作状态

### 布局

- 高度：140
- 横向满宽
- 左侧 65%，右侧 35%
- 背景：极浅蓝灰，不使用图片和渐变
- 圆角：16

### 左侧内容

- 问候语：28 pt semibold
- 当前项目：13 pt secondary
- 当前重点：15 pt medium
- 一句状态摘要：13 pt secondary

### 右侧操作

- 快速记录
- 说入法
- 全局搜索

仅“快速记录”使用主要按钮样式，其余为次级按钮。

## 第二层：核心摘要

使用不完全同质的 4 个摘要模块：

### Agent 当前任务

- 任务名称
- 当前步骤
- 进度
- 暂停 / 打开

### 下一日程

- 突出时间
- 事件名称
- 距离开始时间
- 打开日程

### 待整理内容

- 待整理数量
- 最近来源分布
- 打开收集箱

### 最近剪贴板与 Pin

- 最近 3 条缩略内容
- Pin 数量
- 打开收集箱的剪贴板筛选

### 尺寸

- 4 列
- 列间距 16
- 高度 132
- 最小宽度 220
- 宽度不足时改为 2 × 2

## 第三层：主要工作区

两栏：

```text
左侧 1fr
右侧 320
```

### 左侧：最近活动

- 高度约 330
- 展示 Agent 结果、收集箱处理、日程变更和工具执行
- 每行 56–64
- 支持按项目筛选
- 默认最多展示 6 条

### 右侧：今日计划

- 今日事件
- 今日待办
- 快速操作
- 不使用大量卡片嵌套

## 第四层：系统状态条

固定为压缩横条：

```text
CPU 23% · 内存 38% · 网络 ↑12 KB/s · 电池 82% · 温度 45°C
```

规则：

- 正常状态全部使用中性颜色
- 只有异常项出现语义色
- 点击进入状态页
- 高度 44

## 空状态

首次使用时：

- FocusHeader 仍保留
- 摘要区使用引导状态
- 最近活动提示“内容会从 Agent、收集箱、日程和工具台进入这里”
- 提供一个主要动作：创建第一条记录

## Codex 验收

- 首屏不出现大面积空白
- 系统状态不占据核心位置
- 4 个摘要模块结构不同
- 不使用风景图
- 支持 Loading、Empty、Error
- 1500 × 920 下无需滚动即可看见四层结构
