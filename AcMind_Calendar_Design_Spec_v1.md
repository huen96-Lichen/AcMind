# AcMind 日程表设计规范 v1.0

> 模块定位：**Apple Calendar 式轻量日历 + AcMind 今日执行与复盘面板**  
> 适用范围：AcMind 桌面端 / Electron + React 前端实现  
> 当前版本目标：完成「可用、好看、低心智负担」的日程表第一版

---

## 1. 产品定位

AcMind 日程表不是复杂任务管理器，也不是企业协作日历。

它的核心定位是：

> 一个低干扰、易理解、接近 macOS Calendar 使用心智的个人日历模块，并在右侧提供 AcMind 独有的「今日执行 / 快速记录 / 晚间复盘」能力。

它解决三个问题：

1. **我最近有什么安排？**  
   通过月 / 周 / 日历视图快速查看。

2. **我今天最重要的事情是什么？**  
   通过右侧今日面板聚焦今日重点与当前任务。

3. **今天结束后如何沉淀？**  
   通过快速记录与晚间复盘，把执行过程沉淀到 AcMind 工作台 / Obsidian 体系。

## 2. 与 AcMind 一级导航的关系

AcMind 一级导航建议保持为：

```text
Agent
日程表
工作台
自动工具
设置
```

其中「日程表」负责：

```text
日历查看
时间块安排
今日执行
每日复盘
轻量提醒
```

边界说明：

- 不负责复杂项目管理。
- 不负责知识库沉淀的主界面。
- 不负责批量自动化工具管理。
- 不直接替代 Obsidian。
- 不直接替代完整 Apple Calendar，但可以作为 AcMind 内部的每日节奏入口。

## 3. 设计原则

### 3.1 低心智负担

用户打开日程表后，第一眼应该知道：

- 今天是哪天。
- 当前处于哪个月份 / 周 / 日。
- 哪些日期有安排。
- 今天最重要的事情是什么。

不要让用户先理解复杂系统。

### 3.2 先像日历，再像执行系统

默认心智应接近 macOS Calendar：

```text
左侧分类 + 中央日历 + 顶部切换 + 新建按钮
```

AcMind 的差异化能力放在右侧今日面板中，而不是破坏日历主体。

### 3.3 中央日历是视觉中心

中央月视图 / 周视图 / 日视图必须是页面主焦点。

右侧今日面板是增强能力，不应抢走主视觉。

### 3.4 不做企业后台

禁止出现以下风格：

```text
复杂数据大屏
重型统计卡片
高饱和色块
Jira / Notion 数据库式密集信息
甘特图式项目管理
AI 科技大屏
```

### 3.5 第一版克制

第一版重点是：

```text
看得到
加得上
点得开
改得了
存得住
今天能执行
晚上能复盘
```

不要一开始就堆 AI 自动规划、复杂提醒、拖拽排程、外部日历同步。

## 4. 页面信息架构

整体采用三栏结构：

```text

│ 左侧日历栏 │              中央日历主体              │ 右侧今日面板 │
└──────────────────────────────────────────────────────────────┘
```

推荐尺寸：

| 区域 | 建议尺寸 |

| 顶部栏高度 | 64px - 76px |
| 卡片圆角 | 18px - 24px |
| 弹窗圆角 | 22px - 28px |

## 7. 左侧日历栏规范

### 7.1 模块结构

左侧栏包含：

```text
AcMind 日程表 Logo / 标题

小月历
- 当前月
- 今天高亮
- 上月 / 下月切换

日历分类
- 工作
- 个人
- 项目
- 学习
- 生活
- 自动提醒

其他日历
- 生日
- 节假日

底部用户区
- 头像
- 用户名
- 设置入口
```

### 7.2 小月历要求

小月历用于快速跳转日期。

要求：

1. 显示当前月。
2. 今天使用蓝色圆点高亮。
3. 选中日期使用轻描边或浅蓝底。
4. 点击日期后，中央视图与右侧今日面板同步切换。
5. 上月 / 下月箭头要克制，不要太抢眼。

### 7.3 分类筛选要求

每个分类前有 checkbox 或圆点开关。

交互：

1. 点击分类可以显示 / 隐藏该分类事件。
2. 隐藏分类后，中央日历不显示该分类事件。
3. 分类状态需要本地持久化。
4. 第一版不需要复杂分类管理，但代码应预留后续编辑分类能力。

## 8. 顶部工具栏规范

### 8.1 结构

```text
2026年5月        [ 日 ][ 周 ][ 月 ][ 年 ]        今天  <  >   + 新建   搜索
```

组成：

1. 当前年月标题。
2. 日 / 周 / 月 / 年 segmented control。
3. 今天按钮。
4. 前后切换按钮。
5. 新建按钮。
6. 搜索图标按钮。

### 8.2 交互

- 点击「今天」：跳转到今天，选中今天，并更新右侧今日面板。
- 点击「< / >」：根据当前视图切换上一日 / 周 / 月 / 年。
- 点击「日 / 周 / 月 / 年」：切换中央视图。
- 点击「+ 新建」：打开新建日程弹窗。
- 点击搜索：第一版可只展示搜索输入框或占位弹窗。

### 8.3 视觉

- Segmented control 使用 pill 形态。
- 当前选中项使用浅蓝底 + 蓝色文字。
- 「+ 新建」使用主蓝按钮。
- 其他按钮使用浅灰 / 玻璃按钮。

## 15. 当前任务卡片

### 15.1 目标

让用户一眼知道现在该做什么。

### 15.2 示例

```text
当前任务

14:00 - 15:30
Trae 执行跟进

状态：进行中
检查：构建 / UI / 数据链路

[完成] [延后] [跳过] [交给 Agent]
```

### 15.3 当前任务匹配规则

根据当前时间与事件时间自动判断：

| 规则 | 状态 |
|---|---|
| 当前时间早于开始时间 | 未开始 |
| 当前时间在开始与结束之间 | 进行中 |
| 当前时间晚于结束时间且未操作 | 待确认 / 未完成 |
| 用户手动完成 | 已完成 |
| 用户手动跳过 | 已跳过 |
| 用户手动延后 | 已延后 |

第一版状态可简化为：

```text
未开始
进行中
已完成
已跳过
已延后
```

### 15.4 空状态

```text
现在没有正在进行的日程
可以休息一下，或快速安排下一件事
```

## 23. 数据模型建议

### 23.1 CalendarEvent

```ts
export type CalendarEventStatus =
  | 'not_started'
  | 'in_progress'
  | 'completed'
  | 'skipped'
  | 'postponed'

export type CalendarEventCategory =
  | 'work'
  | 'personal'
  | 'project'
  | 'study'
  | 'life'
  | 'auto_reminder'
  | 'holiday'
  | 'birthday'

export interface CalendarEvent {
  id: string
  title: string
  description?: string
  category: CalendarEventCategory
  status: CalendarEventStatus

  startAt: string
  endAt: string
  allDay?: boolean

  isFocus?: boolean
  reminderAt?: string

  source?: 'manual' | 'agent' | 'imported' | 'system'
  createdAt: string
  updatedAt: string
}
```

### 23.2 DailyFocus

```ts
export interface DailyFocus {
  id: string
  date: string
  items: DailyFocusItem[]
  createdAt: string
  updatedAt: string
}

export interface DailyFocusItem {
  id: string
  title: string
  linkedEventId?: string
  completed?: boolean
}
```

## 30. 最终定性

AcMind 日程表的最终定性：

> AcMind 日程表是一个轻量、低干扰的个人日历模块。它保留用户熟悉的月 / 周 / 日视图与日历分类，同时在右侧提供今日重点、当前任务、快速记录、今日进度和晚间复盘，帮助用户把日程从“记录事项”升级为“管理当天节奏”。

它不是复杂任务管理器。  
它不是企业协作日历。  
它不是 AI 科技大屏。  

它应该像 macOS Calendar 一样简单，但比普通日历更懂「今天该怎么过」。
