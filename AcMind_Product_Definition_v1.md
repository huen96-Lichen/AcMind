# AcMind 产品定型 v1

> **AcMind = Agent 驱动的个人桌面 AI 管家。**  
> 它帮助用户管理时间、沉淀知识、调用工具，并逐步形成自己的个人知识系统。

---

## 1. 一句话定位

**AcMind 是一个以 Agent 为入口的个人桌面 AI 管家，帮助用户管理时间、沉淀知识、调用工具，并逐步形成自己的个人知识系统。**

更短版本：

> **AcMind = Agent 驱动的个人效率与知识中枢。**

---

## 2. 核心产品心智

用户打开 AcMind，不是来管理一堆页面的。

用户打开 AcMind，是为了：

```text
告诉 Agent 我要做什么
↓
安排今天的事情
↓
把信息沉淀进 Obsidian
↓
调用自动工具处理文件、语音、图片和网页
↓
让整个桌面工作流变得更轻松
```

所以 AcMind 的核心心智是：

```text
Agent 管入口
日程表管时间
工作台管知识
自动工具管能力
设置管规则
```

---

## 3. 最终一级架构

AcMind 的一级菜单固定为：

```text
1. Agent
2. 日程表
3. 工作台
4. 自动工具
5. 设置
```

不要再使用以下一级菜单：

- 信息面板
- 工具台
- 能力中心
- 暂存池
- 整理
- 知识库

这些内容应归入对应模块。

---

# 4. 一级模块定义

## 4.1 Agent

### 定位

**默认首页。用户表达意图的入口。**

Agent 是 AcMind 的门面，也是所有功能的自然语言入口。

用户可以通过：

- 文字
- 语音
- 快捷指令

告诉 AcMind 要做什么。

### Agent 承载的功能

```text
Agent
  - 语音输入
  - 文字输入
  - 快捷指令
  - 最近任务
  - 执行反馈
  - 跨模块调度
```

### Agent 能做什么

用户可以说：

```text
帮我安排今天的任务
一个小时后提醒我开会
把今天收集的内容整理一下
把这个 PDF 转成 Markdown
把刚刚这段语音整理成笔记
搜索我之前关于 Acore 的资料
打开 Obsidian 入库内容
检查当前有哪些工具不可用
```

Agent 本身不是所有功能的堆叠区。  
它负责理解意图，然后调用：

```text
日程表
工作台
自动工具
设置
```

---

## 4.2 日程表

### 定位

**个人效率与时间管理平台。**

日程表不是自动工具，不是后台任务中心。

它负责回答：

```text
我今天要做什么？
现在该做什么？
今天完成了什么？
今晚如何复盘？
```

### 日程表承载的功能

```text
日程表
  - 今日
  - 时间轴
  - 日历
  - 任务清单
  - 快速提醒
  - 快速复盘
  - 历史记录
```

### 日程表核心链路

```text
计划 → 执行 → 提醒 → 复盘
```

### 示例功能

| 功能 | 说明 |
|---|---|
| 今日视图 | 展示今天所有任务和提醒 |
| 时间轴 | 按小时安排当天任务 |
| 日历视图 | 查看日 / 周 / 月安排 |
| 任务清单 | 管理待办事项 |
| 快速提醒 | 例如“一小时后提醒我开会” |
| 快速复盘 | 每天结束时总结完成情况 |
| 历史记录 | 查看过去的任务和复盘 |

---

## 4.3 工作台

### 定位

**Obsidian 相关与知识沉淀相关的主流程。**

工作台不是普通任务管理。  
工作台只管一件事：

```text
把零散信息整理成可长期保存的知识。
```

### 工作台承载的功能

```text
工作台
  - 总览
  - 快速入库
  - 暂存区
  - 整理中
  - 待确认
  - 知识库
  - 处理日志
```

### 工作台核心链路

```text
收集 → 暂存 → 整理 → 确认 → 入库 Obsidian
```

### 工作台里的内容

| 模块 | 说明 |
|---|---|
| 总览 | 查看知识处理状态 |
| 快速入库 | 手动文本、文件、网页、语音、截图快速进入流程 |
| 暂存区 | 未整理的原始素材 |
| 整理中 | AI 正在整理的内容 |
| 待确认 | AI 结果等待人工确认 |
| 知识库 | 已沉淀的 Markdown / Obsidian 内容 |
| 处理日志 | 整理、导出、失败记录 |

---

## 4.4 自动工具

### 定位

**当前工具台里的所有工具能力。**

自动工具不是工作台，也不是日程表。  
它负责回答：

```text
AcMind 现在有哪些工具能力可以用？
```

### 自动工具承载的功能

```text
自动工具
  - 工具总览
  - 文件转 Markdown
  - OCR 图像识别
  - 语音转文字
  - 网页正文提取
  - 剪贴板监听
  - 截图捕获
  - 文件夹监听
  - 自动化任务
  - 本地模型
  - 运行状态
```

### 自动工具的边界

自动工具放的是：

```text
能力
工具
依赖
运行状态
测试入口
```

不放：

```text
知识库内容
待确认内容
每日复盘
日程任务列表
```

### 示例归属

| 功能 / 对象 | 归属 |
|---|---|
| PDF 转 Markdown | 自动工具 |
| 某个 PDF 的整理结果 | 工作台 |
| 每天 9 点提醒开会 | 日程表 |
| 创建提醒的自然语言入口 | Agent |
| 是否开启 OCR | 设置 |

---

## 4.5 设置

### 定位

**系统运行规则。**

设置不承担主流程，不放日常操作。

它负责：

```text
AcMind 应该怎么运行？
默认用什么模型？
内容存到哪里？
工具怎么配置？
桌面组件怎么显示？
```

### 设置承载的功能

```text
设置
  - 基础
  - Agent
  - 日程表
  - 工作台
  - 自动工具
  - 桌面组件
  - 语音
  - 知识库
  - 高级
```

### 设置里的关键内容

| 设置项 | 说明 |
|---|---|
| 基础 | 启动、语言、外观、快捷键 |
| Agent | 默认模型、执行策略、确认策略 |
| 日程表 | 提醒方式、默认时间视图、复盘模板 |
| 工作台 | 入库规则、整理规则、导出规则 |
| 自动工具 | OCR、ASR、MarkItDown、Pandoc 等配置 |
| 桌面组件 | 桌面胶囊、顶部 Notch、悬浮输入框 |
| 语音 | 录音设备、转写模型、保存路径 |
| 知识库 | Obsidian Vault、Markdown 模板、Frontmatter |
| 高级 | 数据库、日志、开发者选项、数据迁移 |

---

# 5. 桌面组件归属

桌面胶囊、顶部 Notch、悬浮输入框很重要，但不作为一级菜单。

它们归属：

```text
设置 / 桌面组件
```

同时可以在 Agent 中作为快捷入口出现。

原因是：

```text
桌面组件是交互形态，不是主业务模块。
```

它们负责增强 AcMind 的桌面存在感：

```text
快速唤起
快速输入
快速提醒
快速查看状态
```

---

# 6. 最终产品边界

| 一级菜单 | 管什么 | 不管什么 |
|---|---|---|
| Agent | 意图、语音、指令、调度 | 不直接堆所有页面功能 |
| 日程表 | 时间、任务、提醒、复盘 | 不管 Obsidian 入库 |
| 工作台 | 知识沉淀、Obsidian、入库流程 | 不管工具配置 |
| 自动工具 | 文件、语音、图片、网页等处理能力 | 不展示知识结果 |
| 设置 | 模型、路径、权限、规则 | 不承担日常操作 |

---

# 7. AcMind 的三条产品主线

## 7.1 Agent 管家线

```text
用户表达意图
↓
Agent 理解
↓
调用日程表 / 工作台 / 自动工具
↓
返回执行结果
```

## 7.2 知识沉淀线

```text
收集内容
↓
进入工作台
↓
AI 整理
↓
人工确认
↓
入库 Obsidian
```

## 7.3 个人效率线

```text
制定日程
↓
执行提醒
↓
完成任务
↓
每日复盘
```

这三条线合起来，就是 AcMind 的核心价值。

# 8. 产品调性

AcMind 应该保持：

```text
克制
清晰
本地优先
Apple 风格
Agent-first
桌面原生感
知识沉淀感
个人管家感
```

不要做成：

```text
后台管理系统
普通聊天软件
单纯工具箱
复杂知识库软件
传统日历软件
```

6. 设置保留现有所有设置，并增加日程表、桌面组件等分组。
7. 不删除现有功能，只调整入口、命名、页面职责和导航结构。
8. 页面文案符合新的产品定义。
9. npm run typecheck 通过。
10. npm run build 通过。
