# AcMind Code of Conduct

This project follows the principles of Contributor Covenant 2.1 and is adapted for this repository.

## Our pledge

We are committed to making participation in AcMind a respectful and harassment-free experience for everyone, regardless of background, identity, appearance, experience level, or role.

## Expected behavior

Please help keep the project constructive:

- be considerate and welcoming;
- focus on the work, not on the person;
- give and receive feedback with patience;
- keep discussions specific and evidence-based;
- respect boundaries, privacy, and confidentiality when sensitive information is involved.

## Unacceptable behavior

The following are not acceptable in this project space:

- harassment, intimidation, or discriminatory language;
- doxxing, threats, or sharing private data;
- intentionally disruptive behavior;
- repeated personal attacks or bad-faith escalation.

## Enforcement

Repository maintainers are responsible for interpreting and enforcing this code of conduct.

At the time this file was created, no public moderation email address had been approved for publication. For that reason, the enforcement contact section is intentionally left for maintainer completion before release.

Security-sensitive reports should follow `SECURITY.md` and, when available, GitHub private vulnerability reporting.

## Scope

This code of conduct applies to repository participation, issues, pull requests, comments, and any other project-owned collaboration surface.
