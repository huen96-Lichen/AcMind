# Acore 产品体系地图

> 最后更新：2026-05-05

## 品牌定位

**Acore** 是母品牌，代表个人 AI 系统的核心（AI Core）。

Acore 不是独立功能模块，不作为单独页面出现在应用中。它体现在：
- 应用品牌体系
- 图标命名前缀
- 关于页面
- 产品文档
- 未来多个子产品的统一前缀

## 产品线结构

```
Acore（母品牌）
├── AcMind          主应用 — 个人桌面 AI 信息中枢
├── PinStack        能力层 — AcMind 的桌面快速捕获能力
├── VaultKeeper     引擎层 — AcMind 的内部整理/导入/转换引擎
└── 小龙虾 Agent    未来层 — AcMind 的本地 AI 执行代理
```

## 各产品/模块定位

### AcMind（主应用）

- **定位**：个人桌面 AI 信息中枢
- **形态**：独立桌面应用（macOS）
- **核心职责**：收集 → 暂存 → 整理 → 审阅 → 导出 → 入库
- **用户可见**：是（应用标题、Dock 图标、菜单栏、窗口）

### PinStack（快速捕获层）

- **定位**：AcMind 的桌面快速捕获能力
- **形态**：AcMind 内部能力，不作为独立应用
- **核心职责**：桌面悬浮胶囊、截图捕获、复制即暂存、快速收集入口
- **用户可见**：作为功能名称出现（如"快速捕获"、"桌面胶囊"），不作为独立产品名

### VaultKeeper（整理引擎）

- **定位**：AcMind 的内部整理/导入/转换引擎
- **形态**：内部服务模块
- **核心职责**：文件导入、Markdown 转换、PDF/DOCX/HTML 处理、批量整理、输出规范化
- **用户可见**：不作为独立 App。前台使用"整理"、"导入整理"、"内容转换"等中性语言

### 小龙虾 / Local Agent（未来能力）

- **定位**：AcMind 的本地 AI 执行代理
- **形态**：架构预留，暂不作为一级入口
- **未来职责**：远程对话、本地文件操作、定时整理任务、自动归档
- **用户可见**：当前仅在架构文档中预留
