# AcMind 当前状态说明

## 版本定位

AcMind 已经不是概念验证，而是一个可以持续迭代的 Swift 原生产品骨架。它的核心目标仍然是把零散输入整理为结构化知识，并把结果回写到本地工作流里。

## 当前结构

- `App/` 是应用入口和生命周期编排层
- `AcMindKit/` 是模型、协议、服务和存储层
- `Features/` 是当前活跃的原生界面实现
- `Shared/DesignSystem/` 是共享布局和视觉规范
- 历史迁移残留路径仅作为保留，不应继续扩展

## 已经稳定的部分

- 原生 macOS 主框架已经建立
- 主要工作区、设置页和 Companion 入口已经串起来
- 语音、OCR、剪贴板和文本清洗已经有统一入口
- AI 能力路由已经按能力分层，而不是按单个模型硬编码
- 存储层已经迁移到 SQLite3 封装，不再依赖旧的 Electron 运行时

## 仍需要持续加固的部分

- 启动和退出流程要继续收紧，避免依赖隐式时机
- 自动化调度还需要更完整的可见性和可控性
- Agent、工具面板和长链路任务还在持续模块化
- 文档、工程引用和源码路径要保持一致，避免迁移残留再次扩散

## 当前结论

现在更适合把 AcMind 看成“可运行的产品基座”，而不是已经完全封装好的最终产品。后续工作的重点不是再堆更多入口，而是把现有链路做稳、做清楚、做一致。
