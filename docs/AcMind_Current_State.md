# AcMind 当前状态说明

版本：0.0.1

## 一句话介绍

AcMind 是一款面向 macOS 的原生个人 AI 信息整理管家。它不是单纯聊天工具，而是把用户零散输入收集、清洗、归档、复盘，再按需要提醒和回看的一层本地 AI 工作台。

## 当前定性

当前版本已经从“概念验证”进入“可运行的产品骨架”阶段，整体更接近 alpha / 内测版。

它已经具备了明确的产品方向、稳定的主 UI 框架，以及几条可以完整跑通的核心链路，但距离最终成熟版本仍有明显的产品打磨和自动化完善空间。

## 当前完成度判断

整体完成度约为 65% 到 75%。

这个判断主要基于以下事实：

- 核心导航、设置页、工作台和主要信息流已经存在。
- 语音、OCR、文本清洗、Inbox、Markdown 输出、基础 Provider/模型抽象已经接通。
- AI 能力管理已经从“单一模型调用”升级为“能力分区 + 默认模型 + 保底模型 + Provider 管理”。
- 设置页和主界面已经形成比较完整的原生 macOS 产品形态。

但同时：

- 自动化仍以轻量能力为主。
- 云端复杂任务、Agent、自主执行仍处在预留或半成品状态。
- 跨能力联动、长期记忆、复盘体系、规则策略和失败回退仍需要继续补强。
- 部分功能更偏“工程可用”，离“产品级顺滑体验”还有距离。

## 已完成的核心能力

### 1. 原生 macOS 产品框架

- 主窗口、侧栏、设置页、工作台、任务区的基础结构已建立。
- 整体界面保持原生 SwiftUI 风格，并沿用统一的设计系统。
- 支持当前工作区模式、收起模式和隐藏模式。

### 2. 统一的信息收集入口

AcMind 已经支持从多个来源接收信息：

- 文本输入
- 剪贴板
- 语音
- 截图 / OCR
- 文件
- 页面内容

这些输入会进入统一的信息整理流程，而不是散落在多个互不关联的模块里。

### 3. AI 能力抽象层

项目已经有了清晰的 AI 分层：

- 输入适配
- 任务路由
- 模型能力管理
- 结果处理
- 存储与回写

这意味着后续可以继续接入本地模型、系统框架和云端模型，而不会推翻现有架构。

### 4. 系统原生能力优先

当前已经明确优先使用：

- Apple Vision 做 OCR
- Apple Speech 做语音转文字
- 本地规则/轻量模型做文本清洗和摘要占位

这让 AcMind 的产品方向更偏“本地优先、隐私友好、低依赖”的 macOS 原生工具。

### 5. 设置中心已完成能力化整合

设置页中的 AI 模型区域已经从普通 Provider 列表，升级为：

- 能力分区
- 当前默认模型
- 保底模型
- Provider 管理入口

这一步很关键，因为它把“模型来源”与“功能能力”分开了，后续扩展会更稳。

## 目前仍在继续推进的部分

### 1. 更完整的自动化

当前已有基础提醒和定时能力设计，但还不是完整的自动化系统。

后续仍要补：

- 定时整理 Inbox
- 定时日报 / 周报
- 可见、可暂停、可删除的任务调度
- 更多基于授权的数据回顾

### 2. 更强的复盘与总结

日报和周报已经有雏形，但距离真正“陪伴型 AI 管家”还需要：

- 更稳定的摘要质量
- 更好的任务归纳
- 更清楚的行动建议
- 更强的历史回看能力

### 3. Provider 与云端模型生态

当前已经具备 Provider 抽象，但云端复杂任务并不是默认路径。

后续目标是：

- 本地优先
- 云端按需
- 模型不可用时自动回退
- 所有能力可视、可控、可关闭

## 产品定位总结

AcMind 不是 ChatGPT 替代品，而是一个常驻 macOS 桌面的个人信息整理层。

它更像是：

- 个人输入的收集器
- 资料的清洗器
- 任务的归纳器
- 日报和复盘的生成器
- 下一步行动的提醒器

## 适合对外介绍的表述

如果要对外快速介绍当前 AcMind，可以这样说：

> AcMind 是一款面向 macOS 的原生个人 AI 管家，帮助用户把语音、截图、文本、文件和剪贴板中的零散信息统一收集、清洗、整理、归档，并在合适的时候生成日报、复盘和提醒。

## 当前版本的真实状态

这是一个已经具备明确产品方向和主链路能力的早期版本。

它适合：

- 演示产品方向
- 继续做核心链路迭代
- 作为后续版本的稳定基座

它暂时还不适合：

- 直接当作完全成熟的生产级 AI 中台
- 依赖大量自动化却不加约束地全量接管电脑行为

## 下一个阶段建议

下一阶段优先级建议是：

1. 固化核心链路的体验和稳定性。
2. 提升日报 / 复盘的质量。
3. 继续完善本地优先的能力路由。
4. 再扩展云端复杂任务和 Agent 能力。
