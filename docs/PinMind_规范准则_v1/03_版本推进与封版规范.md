# AcMind 版本推进与封版规范 v1

## 1. 基本原则

AcMind 每个版本只能有一个主目标。

禁止一个版本同时推进多个大方向。

错误示例：

```txt
v0.2 同时做 AI 面板、Obsidian、截图、网页剪藏、模型市场、设置页、美化、搜索、知识图谱
```

正确示例：

```txt
v0.2 只完成“信息捕获进入本地 Inbox”
```

## 2. 版本文档模板

每个版本必须有独立任务文档：

```md
# AcMind vX.X.X 版本任务

## Version Goal
本版本唯一目标是什么？

## Scope
本版本做什么？

## Out of Scope
本版本明确不做什么？

## Technical Plan
技术实现方案是什么？

## UI Plan
界面和交互怎么做？

## Data Plan
涉及哪些数据结构和持久化？

## Acceptance
怎样才算完成？

## Risk
可能出现什么问题？

## Required Docs Update
完成后必须更新哪些文档？
```

## 3. 推荐早期版本路线

### v0.1 项目骨架与品牌重建

目标：

> 从 PinStack 派生出 AcMind 独立项目，完成基础命名、结构、视觉方向、版本系统。

必须完成：

- 新项目名 AcMind
- package 信息改名
- 应用内版本号
- 基础路由
- 主窗口
- 设置页骨架
- 本地数据目录
- 日志目录
- 基础设计 token
- README
- CHANGELOG
- PROJECT_HANDOVER

不要求完成 AI。

### v0.2 本地 Inbox 信息捕获

目标：

> 任意文本可以进入 AcMind 的本地 Inbox。

必须完成：

- 手动输入文本
- 剪贴板捕获文本
- 本地保存原始内容
- Inbox 列表
- 详情预览
- 删除 / 归档 / 标记状态
- 基础日志

验收标准：

- 重启后数据仍在
- 捕获失败有提示
- 本地文件可追踪
- 不依赖云端

### v0.3 AI 蒸馏链路 Mock 版

目标：

> 不依赖真实模型，先跑通“原始内容 → AI 整理结果”的产品链路。

必须完成：

- AI 任务队列
- Mock provider
- 总结
- 改标题
- 提取标签
- 提取摘要
- 输出 Markdown 预览
- 任务状态：pending / running / success / failed

验收标准：

- 即使没有模型，也能演示完整流程
- AI 处理过程有状态
- 失败能重试
- 结果能保存

### v0.4 Obsidian 输出闭环

目标：

> AI 整理后的内容可以输出到 iCloud 同步的 Obsidian Vault。

必须完成：

- Vault 路径设置
- 输出目录设置
- Markdown 命名规范
- frontmatter
- tags
- source 字段
- created / updated 字段
- 原文保留策略
- 输出成功提示
- 打开文件位置

验收标准：

- Obsidian 能直接识别
- Markdown 格式稳定
- 不破坏双链
- 不破坏 frontmatter
- 不覆盖旧文件

### v0.5 真实 AI Provider 接入

目标：

> 接入本地模型和云端模型，但模型可插拔、可关闭、可降级。

必须完成：

- 本地模型 Provider
- 云端模型 Provider
- Provider 设置页
- API Key 管理
- 模型启用 / 禁用
- 任务级模型选择
- 失败降级
- 消耗提示
- 日志记录

验收标准：

- 没有 API Key 不崩溃
- 本地模型不可用时能回退
- 云端失败时有明确提示
- 用户知道当前用了哪个模型等级

### v0.6 AcMind 主控制台

目标：

> 用户打开软件后，能直观看到今天捕获了什么、处理了什么、沉淀了什么。

必须完成：

- Dashboard
- 今日捕获
- 待处理
- 已蒸馏
- 已归档
- AI 任务状态
- 模型运行状态
- 快捷入口

验收标准：

- 用户一眼能理解软件在做什么
- 信息密度高但不拥挤
- 每个卡片都有真实数据
- 不再出现“点了没反应”

## 4. 封版条件

一个版本必须同时满足：

- 功能完成
- UI 可用
- 设置持久化
- 数据可恢复
- 错误有反馈
- 日志可追踪
- README / CHANGELOG / PROJECT_HANDOVER 已更新
- Trae 已输出交付报告
- Codex 已通过核验

否则不能封版。

## 5. 版本完成后的必更文档

每个版本完成后必须更新：

```txt
README.md
CHANGELOG.md
docs/PROJECT_HANDOVER.md
docs/ROADMAP.md
docs/KNOWN_ISSUES.md
```

涉及 AI 时额外更新：

```txt
docs/AI_PROVIDER_SPEC.md
docs/AI_TASK_SPEC.md
```

涉及 Obsidian 时额外更新：

```txt
docs/OBSIDIAN_OUTPUT_SPEC.md
```

涉及 UI 时额外更新：

```txt
docs/UI_DESIGN_SYSTEM.md
```
