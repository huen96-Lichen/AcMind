# AcMind 代码架构与数据模型规范 v1

## 1. 架构基本原则

AcMind 是本地优先软件。

所以架构设计必须遵守：

- 本地数据可见
- 本地数据可备份
- 本地数据可迁移
- 云端只是增强，不是依赖
- AI Provider 可插拔
- Obsidian 输出稳定
- 核心数据模型少变

## 2. 核心数据对象

早期建议固定几个核心对象：

- SourceItem：所有进入 AcMind 的原始信息
- AiTask：AI 处理任务
- DistilledOutput：AI 蒸馏后的结果
- ExportRecord：输出到 Obsidian 的记录
- AppSettings：应用设置
- ProviderConfig：模型供应商配置

## 3. SourceItem

SourceItem 是 AcMind 的核心对象。

所有信息都先变成 SourceItem，再进入后续流程。

```ts
type SourceType =
  | 'clipboard'
  | 'manual'
  | 'screenshot'
  | 'image'
  | 'pdf'
  | 'web'
  | 'audio'
  | 'video';

type SourceStatus =
  | 'captured'
  | 'queued'
  | 'processing'
  | 'distilled'
  | 'archived'
  | 'failed';

interface SourceItem {
  id: string;
  type: SourceType;
  title: string;
  rawText?: string;
  rawFilePath?: string;
  createdAt: string;
  updatedAt: string;
  status: SourceStatus;
  tags: string[];
  summary?: string;
  distilledMarkdownPath?: string;
  obsidianPath?: string;
}
```

## 4. AiTask

AI 任务必须队列化，不允许 UI 点击后直接同步请求模型。

```ts
type AiTaskStatus =
  | 'pending'
  | 'running'
  | 'success'
  | 'failed'
  | 'cancelled';

type AiTaskType =
  | 'summarize'
  | 'rename'
  | 'tag'
  | 'clean'
  | 'distill'
  | 'export';

type AiProviderTier =
  | 'local_light'
  | 'cloud_standard'
  | 'cloud_strong';

interface AiTask {
  id: string;
  sourceId: string;
  type: AiTaskType;
  status: AiTaskStatus;
  providerTier: AiProviderTier;
  createdAt: string;
  startedAt?: string;
  finishedAt?: string;
  error?: string;
}
```

## 5. DistilledOutput

```ts
interface DistilledOutput {
  id: string;

  summary: string;
  tags: string[];
  markdown: string;
  createdAt: string;
  updatedAt: string;

```

## 6. ExportRecord

```ts
interface ExportRecord {
  id: string;
  sourceId: string;
  outputId: string;
  target: 'obsidian' | 'local_markdown';
  filePath: string;
  createdAt: string;
  status: 'success' | 'failed';
  error?: string;
}
```

interface AppSettings {

## 8. 本地文件结构

建议本地数据目录：

```txt
AcMind/
├── data/
│   ├── sources/
│   ├── tasks/
│   ├── outputs/
│   ├── settings/
│   └── index/
├── logs/
│   ├── app.log
│   ├── ai.log
│   ├── export.log
│   └── error.log
└── backups/
```

## 9. 数据存储约束

禁止：

- 所有数据塞进一个巨大 JSON
- 原始内容只存在内存中
- AI 结果覆盖原文
- 设置只保存在 UI state
- 文件路径写死
- 用户数据保存到项目源码目录

必须：

- 原文保留
- AI 输出单独保存
- 设置持久化
- 任务记录可恢复
- 输出记录可追踪
- 错误日志可查看

## 10. 迁移约束

如果修改核心数据结构，必须提供：

- 旧字段
- 新字段
- 迁移逻辑
- 兼容策略
- 失败回滚方式
- CHANGELOG 说明

## 11. 主链路约束

所有主流程应该围绕 SourceItem 展开：

```txt
SourceItem
↓
AiTask
↓
DistilledOutput
↓
ExportRecord
```

不要让各功能绕开 SourceItem 单独实现自己的数据链路。
