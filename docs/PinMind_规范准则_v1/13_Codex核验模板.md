# AcMind Codex 核验模板

> 使用方式：每次 Trae 完成任务后，把任务说明、Trae 交付报告和本模板一起交给 Codex。

# AcMind vX.X.X Codex 核验请求

## 一、核验目标

请对 Trae 本次交付进行只读核验，判断是否真正完成任务要求。

不要只看 Trae 的文字说明，请以代码事实为准。

## 二、任务原始要求

在这里粘贴 Trae 的任务说明。

## 三、Trae 交付报告

在这里粘贴 Trae 的交付报告。

## 四、必须重点核验

请重点检查：

1. 功能是否真实接入主链路
2. UI 是否只是摆设
3. 设置是否持久化
4. 重启后数据是否仍然存在
5. Mock 是否明确标注
6. 是否存在假功能
7. 是否破坏已有功能
8. 是否有类型问题
9. 是否有日志
10. 是否更新必要文档

## 五、核验报告格式

请严格按以下格式输出：

```md
## 结论
通过 / 部分通过 / 不通过

## 真实完成项
- ...

## 假完成项
- ...

## 主要风险
- ...

## 必须修复
- ...

## 建议修复
- ...

## 是否允许进入下一版本
是 / 否
```

## 六、质量门禁

如果出现以下任一问题，请不要判定通过：

- UI 有按钮但无逻辑
- 设置不能持久化
- 重启后数据丢失
- Mock 冒充真实功能
- 主链路被绕开
- 错误无提示
- 日志无记录
- 页面不能滚动导致信息被遮挡
- 版本文档未更新
