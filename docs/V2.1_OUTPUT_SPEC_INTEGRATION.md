# AcMind V2.1 OutputSpec 集成文档

## 概述

本文档描述 AcMind V2.1 Phase 0 中 OutputSpecService 的设计与集成方式。

## 设计原则

1. **模板不硬编码**：所有 Markdown 模板从 `acmind_output_spec_pack` 读取，业务代码不包含模板字符串。
2. **Fallback 保障**：模板包缺失时，服务自动降级到内置默认值，不影响核心功能。
3. **三层分离**：标准字段层（语义）→ Format Profile 层（格式）→ Markdown 文件层（存储）。

## OutputSpecService

### 文件位置

```
src/main/services/outputSpec/
  index.ts              # 导出入口
  outputSpecService.ts  # 核心服务
```

### 初始化

在 `src/main/index.ts` 的 `bootstrap()` 中初始化：

```typescript
outputSpecService.init(specPackPath);
```

服务会按以下优先级查找模板包：
1. `{app.getPath()}/acmind_output_spec_pack`
2. `{process.cwd()}/acmind_output_spec_pack`
3. 用户指定路径（开发环境）

### 核心能力

| 方法 | 说明 |
|------|------|
| `getActiveProfile()` | 获取当前活跃的 Format Profile |
| `getProfile(id)` | 按 ID 获取指定 Profile |
| `getDefaultTemplate()` | 获取默认 Markdown 模板 |
| `getTemplate(name)` | 按名称获取模板（default / with-raw-content / manual-minimal） |
| `getTagRules()` | 获取标签规则 |
| `getCategoryRules()` | 获取分类规则 |
| `getDefaultFields()` | 获取默认标准字段 |
| `getSpecPackInfo()` | 获取规范包加载状态信息 |

### 模板优先级

`MarkdownBuilder.build()` 的模板选择优先级：

1. 显式传入的 `template` 参数
2. `VaultConfig.template`（用户自定义模板）
3. `OutputSpecService.getDefaultTemplate()`（从规范包读取）
4. `LEGACY_DEFAULT_TEMPLATE`（终极 fallback）

### IPC 通道

| 通道 | 说明 |
|------|------|
| `outputSpec.getInfo` | 获取规范包加载信息 |
| `outputSpec.getActiveProfile` | 获取活跃 Profile |
| `outputSpec.getProfile` | 按 ID 获取 Profile |
| `outputSpec.getTemplate` | 获取指定模板 |
| `outputSpec.getTagRules` | 获取标签规则 |
| `outputSpec.getCategoryRules` | 获取分类规则 |

## 模板包结构

```
acmind_output_spec_pack/
  02_模板/
    默认 Markdown 模板.md           → TemplateName: 'default'
    带原始内容 Markdown 模板.md     → TemplateName: 'with-raw-content'
    人工最小维护模板.md             → TemplateName: 'manual-minimal'
  03_Format_Profile/
    acmind-default.profile.json    → Format Profile 配置
  04_人工维护/
    标签与分类维护建议.md           → 标签/分类规则
```

## 受影响的模块

| 模块 | 变更 |
|------|------|
| `markdownBuilder.ts` | 模板来源改为 OutputSpecService，新增 `buildFromFields()` 方法 |
| `obsidianExporter.ts` | Profile 来源改为 OutputSpecService |
| `index.ts` (bootstrap) | 新增 OutputSpecService 初始化 |
| `ipc.ts` | 新增 6 个 OutputSpec IPC 通道 |
| `types.ts` | 新增 IPC_CHANNELS 定义 |
