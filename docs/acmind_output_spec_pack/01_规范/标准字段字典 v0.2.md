# AcMind 标准字段字典 v0.2

## 1. 字段设计原则

AcMind 内部使用英文 key，用户界面和文档说明使用中文名称。

这样做的原因：

- 英文 key 更适合 TypeScript、数据库、JSON、迁移脚本
- 中文名称更适合普通用户理解
- 模型可以稳定输出英文 key，也可以在界面层展示中文
- 后续更换模板时不需要改变字段语义

---

## 2. 字段总表

| 中文名称 | 内部 key | 类型 | 必填 | 默认值 | 说明 |
|---|---|---:|---:|---|---|
| 标题 | `title` | string | 是 | 无 | 文档主名称 |
| 总结 | `summary` | string | 是 | 无 | 一句话总结 |
| 标签 | `tags` | string[] | 否 | `[]` | 检索和聚合用 |
| 分类 | `category` | string | 否 | `未分类` | 主归类，只能一个 |
| 正文 | `body` | string | 是 | 无 | 整理后的正文 |
| 原始内容 | `raw_content` | string | 否 | 空 | 原始输入内容 |
| 来源 | `source` | string | 是 | `manual` | 内容来源 |
| 采集时间 | `captured_at` | string | 是 | 当前时间 | 采集时间 |
| 项目 | `project` | string | 否 | `默认` | 所属项目 |
| 状态 | `status` | string | 是 | `collected` | 流程状态 |
| 置信度 | `confidence` | number | 否 | `0.5` | 模型可信度 |
| 格式版本 | `schema_version` | string | 是 | `0.2` | 字段规范版本 |

---

## 3. 状态枚举

| 中文展示 | 内部值 | 说明 |
|---|---|---|
| 已收集 | `collected` | 内容进入系统，但未清洗 |
| 已清洗 | `cleaned` | 已经过模型或规则清洗 |
| 待审阅 | `reviewing` | 等待用户确认 |
| 已审阅 | `reviewed` | 用户已经确认 |
| 已导出 | `exported` | 已输出为 Markdown |
| 失败 | `failed` | 处理失败，需要重试 |

状态值必须固定，不允许随意写自然语言。

---

## 4. 来源枚举

| 中文展示 | 内部值 |
|---|---|
| 剪贴板 | `clipboard` |
| 截图 | `screenshot` |
| 网页 | `webpage` |
| 手动输入 | `manual` |
| 文件导入 | `file` |
| 语音转写 | `transcript` |
| 图片识别 | `image_ocr` |
| 其他 | `other` |

来源必须真实、可追踪。模型不能编造来源。

---

## 5. 置信度解释

| 数值范围 | 含义 | 建议动作 |
|---|---|---|
| `0.9 ~ 1.0` | 高可信 | 可直接使用 |
| `0.7 ~ 0.89` | 基本可信 | 建议快速浏览 |
| `0.5 ~ 0.69` | 中等可信 | 建议审阅 |
| `0 ~ 0.49` | 低可信 | 建议重新处理 |

如果模型不确定，应降低 `confidence`，不要假装确定。

---

## 6. 默认字段对象

```json
{
  "schema_version": "0.2",
  "title": "",
  "summary": "",
  "tags": [],
  "category": "未分类",
  "body": "",

  "source": "manual",
  "captured_at": "2026-04-30 10:34",
  "project": "默认",
  "status": "collected",
  "confidence": 0.5
}
```
