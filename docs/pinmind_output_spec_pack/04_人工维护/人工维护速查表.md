# PinMind 人工维护速查表

这份文档给普通用户使用。即使没有 AI，也可以根据这里维护 PinMind 导出的 Markdown。

---

## 1. 我只想快速新建一篇

复制这个模板：

```markdown
---
schema_version: "0.2"
title: "标题"
summary: "一句话总结"
tags: []
category: "未分类"
source: "manual"
captured_at: "2026-04-30 10:34"
project: "默认"
status: "collected"
confidence: 0.5
---

> 一句话总结。

# 标题

正文内容。
```

---

## 2. 我只想改标题

需要同步改三个地方：

1. 文件名
2. frontmatter 里的 `title`
3. 正文里的一级标题

示例：

```markdown
---
title: "新的标题"
---

# 新的标题
```

---

## 3. 我只想改标签

改这里：

```yaml
tags:
  - 标签一
  - 标签二
```

建议：

- 标签可以多个
- 优先中文
- 不要同义重复
- 不要每篇都加十几个标签

---

## 4. 我只想改分类

改这里：

```yaml
category: "未分类"
```

分类只写一个。

复杂关系交给标签，不要写成：

```yaml
category: "产品 / AI / Obsidian / Markdown / 项目"
```

---

## 5. 我只想改项目

改这里：

```yaml
project: "默认"
```

例如：

```yaml
project: "PinMind"
```

---

## 6. 我只想标记已经整理完

改这里：

```yaml
status: "reviewed"
```

---

## 7. 我只想标记已经导出

改这里：

```yaml
status: "exported"
```

---

## 8. 常用状态

| 想表达 | 写法 |
|---|---|
| 刚收集 | `collected` |
| 已清洗 | `cleaned` |
| 待审阅 | `reviewing` |
| 已审阅 | `reviewed` |
| 已导出 | `exported` |
| 处理失败 | `failed` |

---

## 9. 维护原则

- 标题要短而清楚。
- 总结只写 1 到 2 句话。
- 标签可以多个，分类只能一个。
- 正文可以自由写，但不要删除重要信息。
- 如果不确定分类，写 `未分类`。
- 如果不确定项目，写 `默认`。
- 如果不确定状态，写 `collected`。
