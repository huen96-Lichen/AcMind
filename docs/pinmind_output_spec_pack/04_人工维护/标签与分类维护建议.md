# PinMind 标签与分类维护建议

## 1. 标签和分类的区别

分类是主归属。

标签是辅助索引。

简单理解：

```text
分类：这篇内容主要属于哪里？
标签：这篇内容还和哪些概念有关？
```

---

## 2. 分类规则

分类只保留一个。

推荐分类：

```text
产品规范
项目记录
学习笔记
灵感想法
工作资料
个人复盘
技术文档
素材资料
未分类
```

不要过度细分。

---

## 3. 标签规则

标签可以多个。

推荐标签写法：

```yaml
tags:
  - PinMind
  - Obsidian
  - Markdown
```

不推荐：

```yaml
tags:
  - pinmind
  - Pinmind
  - pin-mind
  - 我的AI软件
```

同一概念应该尽量保持同一个标签。

---

## 4. 标签命名建议

建议：

- 优先中文或固定产品名
- 产品名保持官方写法
- 不要中英文混乱
- 不要创建太长标签
- 不要用整句话当标签

---

## 5. 什么时候新增标签

适合新增标签的情况：

- 这个概念以后会反复出现
- 这个主题值得单独聚合
- 这个标签能帮助你未来搜索
- 这个标签能连接多个项目或文档

不适合新增标签的情况：

- 只在这篇文档出现一次
- 只是临时形容词
- 和已有标签意思差不多
- 太细导致以后维护困难
