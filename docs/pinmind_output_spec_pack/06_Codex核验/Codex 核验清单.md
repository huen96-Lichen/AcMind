# PinMind 默认输出规范 - Codex 核验清单

## 1. 核验目标

检查 Trae 是否正确实现 PinMind 默认输出系统。

重点确认：

- 标准字段层是否独立
- Format Profile 是否只负责渲染
- Markdown 是否统一生成
- 用户是否可以脱离 AI 手动维护
- Obsidian 是否能正常承接

---

## 2. 字段层核验

请检查：

- [ ] 是否存在统一标准字段类型 `PinMindStandardFields`
- [ ] 是否包含 `schema_version`
- [ ] 是否包含 `title`
- [ ] 是否包含 `summary`
- [ ] 是否包含 `tags`
- [ ] 是否包含 `category`
- [ ] 是否包含 `body`
- [ ] 是否支持 `raw_content`
- [ ] 是否包含 `source`
- [ ] 是否包含 `captured_at`
- [ ] 是否包含 `project`
- [ ] 是否包含 `status`
- [ ] 是否包含 `confidence`
- [ ] 是否有默认值填充逻辑
- [ ] 是否区分内部英文 key 和用户展示中文名称

---

## 3. 状态枚举核验

请检查状态是否固定为：

- [ ] `collected`
- [ ] `cleaned`
- [ ] `reviewing`
- [ ] `reviewed`
- [ ] `exported`
- [ ] `failed`

不允许状态是任意自然语言。

---

## 4. 渲染层核验

请检查：

- [ ] 是否存在 MarkdownRenderer
- [ ] Renderer 输入是否为 `标准字段 + Format Profile`
- [ ] 模型是否没有直接输出最终 Markdown
- [ ] 默认 Profile 是否可以生成合法 Markdown
- [ ] 是否可以通过更换 Profile 重新渲染
- [ ] Profile 是否没有参与内容理解

---

## 5. Obsidian 兼容核验

请检查导出的 Markdown：

- [ ] frontmatter 是否合法
- [ ] tags 是否为 Obsidian 可识别格式
- [ ] 文件名是否安全
- [ ] 日期格式是否统一
- [ ] 正文是否可直接阅读
- [ ] 没有 PinMind 时也能打开维护

---

## 6. 文件命名核验

请检查：

- [ ] 默认文件名是否为 `YYYY-MM-DD HH-mm - 标题.md`
- [ ] 是否清洗 `/ \ : * ? " < > |`
- [ ] 文件名是否不会过长
- [ ] 文件名是否和标题保持基本一致

---

## 7. 迁移层核验

如果已实现迁移功能，请检查：

- [ ] 是否支持仅影响新内容
- [ ] 是否支持整个 vault 迁移
- [ ] 是否支持某个 project 迁移
- [ ] 迁移是否只重新渲染模板
- [ ] 迁移是否不改变标准字段语义
- [ ] 迁移前是否生成备份
- [ ] 迁移前是否支持预览
- [ ] 迁移后是否生成日志
- [ ] 是否支持回滚

---

## 8. 人工维护核验

请检查导出内容是否满足：

- [ ] 用户能看懂标题
- [ ] 用户能看懂总结
- [ ] 用户能手动修改标签
- [ ] 用户能手动修改分类
- [ ] 用户能手动修改正文
- [ ] 用户能手动修改项目
- [ ] 用户能在 Obsidian 里直接维护

---

## 9. 反模式检查

请重点排查是否存在以下问题：

- [ ] 模型直接输出最终 Markdown
- [ ] Format Profile 参与内容语义判断
- [ ] 分类允许多个
- [ ] 状态允许自由文本
- [ ] 标签生成过多且无约束
- [ ] 批量迁移没有备份
- [ ] raw_content 被默认删除且无法回溯
- [ ] 文件名没有清洗危险字符
