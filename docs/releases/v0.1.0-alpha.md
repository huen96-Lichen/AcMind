# v0.1.0-alpha Release Preparation

This document prepares, but does not publish, the first alpha release.

## Release summary

AcMind is not yet a fully polished public release. The alpha should present the project as a credible, reproducible, security-conscious pre-release macOS workspace with known test limitations that are documented openly.

## Verified installation path

Users should install from source with the documented build paths in `docs/testing-and-build-baseline.md`.

Verified commands include:

```bash
swift package resolve
swift build
xcodebuild \
  -project AcMind.xcodeproj \
  -scheme AcMind \
  -configuration Debug \
  -destination 'platform=macOS' \
  CODE_SIGNING_ALLOWED=NO \
  CODE_SIGNING_REQUIRED=NO \
  build
```

The repository also has a release-oriented build script. Its package mode may require `DEVELOPER_ID=-` on machines without a Developer ID certificate.

## Known limitations

- `swift test` still has documented known failures in the baseline document.
- The project remains pre-release.
- Permission-heavy features depend on user consent and macOS privacy settings.
- Cloud-backed workflows can send user-selected content off device.
- Release packaging should be checked carefully on a clean machine before publishing.

## Checksum generation

Generate checksums after the release artifact is built and before publishing:

```bash
shasum -a 256 path/to/AcMind.dmg
```

Record the checksum in the release notes or attached release checklist.
