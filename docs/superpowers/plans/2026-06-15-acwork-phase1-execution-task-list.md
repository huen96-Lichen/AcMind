# AcWork Phase 1 执行任务单

> 目的：把 `615设计规范` 中的核心要求整理成可执行、可验收、可排序的任务单，供本轮重制直接推进。  
> 范围：只覆盖 Phase 1 的三块核心交付物
> - 统一应用外壳
> - 工作台
> - 合并后的收集箱
>
> 设计基准：1500 × 920 pt  
> 最小窗口：1180 × 720 pt  
> 响应阈值：1320 pt

## 0. 先决条件

- [ ] 固定当前基线，保留未完成改动，不回滚用户工作。
- [ ] 记录构建结果、测试结果、窗口尺寸、页面现状和旧数据路径。
- [ ] 准备三张核心页面的截图基线：工作台、收集箱列表、收集箱网格。
- [ ] 确认 Phase 1 只做三类核心屏幕，不扩展到 Agent、日程、工具台等页面重制。

## 1. 信息架构与品牌迁移

### 1.1 用户可见命名统一为 AcWork

- [ ] 应用标题、菜单、欢迎文案、关于页、设置页、通知展示文案统一改为 `AcWork`。
- [ ] 保留 `AcMindKit`、Bundle ID、数据库目录、旧通知原值等兼容项。
- [ ] 区分“用户可见名称”和“技术兼容标识”，避免机械全局替换。

### 1.2 一级导航重构

- [ ] 固定 10 个一级导航项：工作台、Agent、收集箱、日程、工具台、灵动大陆、说入法、状态、模型、设置。
- [ ] 移除独立的“剪贴板”一级页面入口。
- [ ] 保留旧 `clipboard` 路由兼容，但最终落到收集箱剪贴板筛选。
- [ ] 统一导航顺序、分组和快捷键，避免 `Cmd+2` 冲突。
- [ ] 旧持久化选中值增加 fallback，未知值回到工作台。

### 1.3 窗口策略

- [ ] 主窗口默认尺寸改为 1500 × 920。
- [ ] 主窗口最小内容尺寸改为 1180 × 720。
- [ ] 历史窗口恢复时，对小于最小尺寸的窗口做校正。
- [ ] 屏幕可用区域不足时，保持窗口可见，不允许屏幕外定位。

## 2. 统一应用外壳

### 2.1 Shell 结构

- [ ] 固定 Sidebar 宽度为 216 pt。
- [ ] 固定 Toolbar 高度为 60 pt。
- [ ] Page Body 支持可选 Filter Rail、Main Content、Inspector。
- [ ] 页面切换时不替换根布局，不引起整体尺寸跳动。
- [ ] 页面标题只能在 Toolbar 出现一次，正文不重复。

### 2.2 Sidebar

- [ ] 品牌区高度 72 pt。
- [ ] 导航项高度 38 pt，圆角 10 pt。
- [ ] 图标尺寸 16-18 pt。
- [ ] 分组标题 11 pt semibold。
- [ ] 选中态为浅蓝背景 + 系统蓝图标。
- [ ] Hover 为浅灰背景。
- [ ] 不永久显示快捷键标签，Hover 时可显示。
- [ ] 底部显示本地服务状态和当前模型。
- [ ] 支持键盘上下移动与激活。

### 2.3 Toolbar

- [ ] 只承载当前页面上下文，不重复内容区标题。
- [ ] 支持 title、context、primaryAction、secondaryActions、search 插槽。
- [ ] 页面主操作与搜索集中到统一工具栏。

### 2.4 响应式

- [ ] 1320 pt 及以上允许三栏布局。
- [ ] 1180-1319 pt 隐藏固定 Inspector，改用 Sheet 或 Overlay。
- [ ] 高度低于 760 pt 时收紧垂直间距，不隐藏核心操作。
- [ ] 常规卡片不使用明显阴影，仅保留 1 pt 描边。

## 3. Design Tokens 与统一组件

### 3.1 Tokens 收口

- [ ] 建立主窗口唯一 Token 来源。
- [ ] 明确定义 Layout：Sidebar 216、Toolbar 60、Padding 20、Gap 16、Filter Rail 220、Inspector 304。
- [ ] 明确定义 Radius、Spacing、Typography、语义色。
- [ ] 主窗口优先使用系统语义色，保证深浅色和高对比度可读。
- [ ] 旧 Token 只作为兼容层，禁止继续新增页面级 Token。

### 3.2 基础组件

- [ ] 实现 `AcWorkShell`。
- [ ] 实现 `AcSidebar`。
- [ ] 实现 `AcPageToolbar`。
- [ ] 实现 `AcSection`。
- [ ] 实现 `AcCard`。
- [ ] 实现 `AcListRow`。
- [ ] 实现 `AcInspector`。
- [ ] 实现 `AcStatusBadge`。
- [ ] 实现 `AcMetric`。
- [ ] 实现 `AcEmptyState`。
- [ ] 实现 `AcSearchField`。
- [ ] 实现 `AcSegmentedControl`。
- [ ] 实现 `AcActionButton`。
- [ ] 实现 `AcSettingRow`。
- [ ] 实现 `AcPermissionRow`。

### 3.3 统一状态

- [ ] 每个核心组件覆盖 Normal、Hover、Selected、Disabled、Keyboard Focus。
- [ ] 页面级组件覆盖 Loading、Empty、Error、Permission Required。
- [ ] 组件通过显式参数和 Binding 接收状态，不直接读取整棵 AppState。
- [ ] 空状态最多保留一个主要操作。

## 4. 工作台

### 4.1 数据契约

- [ ] 定义工作台 ViewModel 的数据模型。
- [ ] 定义 Repository 协议。
- [ ] 使用真实 Service 组合数据。
- [ ] 提供 Mock Repository。
- [ ] ViewModel 明确输出 Loading、Loaded、Empty、Error。

### 4.2 页面结构

- [ ] 实现 Focus Header。
- [ ] 实现 4 个差异化摘要模块。
- [ ] 实现最近活动与今日计划两栏布局。
- [ ] 实现压缩系统状态条。
- [ ] 首次使用时提供明确的 Empty 引导。

### 4.3 工作台内容要求

- [ ] 当前工作状态回答“现在在做什么”。
- [ ] 下一日程回答“接下来做什么”。
- [ ] 待整理内容回答“哪些内容等处理”。
- [ ] 最近剪贴板与 Pin 回答“最近捕获了什么”。
- [ ] 系统状态只作为压缩辅助信息，不成为页面核心。

## 5. 收集箱统一模型

### 5.1 统一领域模型

- [ ] 定义 `CollectedItem`。
- [ ] 定义 `CollectionSource`。
- [ ] 定义 `CollectedContentType`。
- [ ] 定义 `ProcessingStatus`。
- [ ] 保证 `SourceItem` 与 `ClipboardItem` 的 ID 不冲突。
- [ ] 保证内容类型覆盖文本、链接、图片、文件、代码、富文本、视频。

### 5.2 Adapter

- [ ] `SourceItem` 映射到 `CollectedItem`。
- [ ] `ClipboardItem` 映射到 `CollectedItem`。
- [ ] 保留标题、预览、来源、时间、类型、标签等关键字段。
- [ ] 图片和文件只传递资源引用，不在列表层加载完整大文件。

### 5.3 Repository

- [ ] 建立 `CollectedItemRepository` 聚合层。
- [ ] 聚合 `StorageServiceProtocol` 与 `ClipboardServiceProtocol`。
- [ ] 支持列表、搜索、筛选、排序、分页或限制数量。
- [ ] 支持更新标签、Pin、收藏、归档、删除。
- [ ] 支持保存到知识库、发送给 Agent、转任务、转日程、导出 Markdown。
- [ ] 支持粘贴队列读取和操作。

## 6. 收集箱 UI

### 6.1 左侧 Filter Rail

- [ ] 固定宽度 220 pt。
- [ ] 支持快捷视图：全部、待整理、已 Pin、已收藏、最近使用。
- [ ] 支持来源筛选：剪贴板、手机同步、说入法、截图与 OCR、Agent、手动添加。
- [ ] 支持类型筛选：文本、链接、图片、文件、代码、富文本、视频。
- [ ] 支持状态筛选：待整理、已提炼、已归档、已导出。
- [ ] 支持多条件组合与分项数量。
- [ ] 筛选变化时不重建整个页面。

### 6.2 顶部操作栏

- [ ] 显示当前筛选摘要。
- [ ] 提供搜索。
- [ ] 提供排序。
- [ ] 提供列表 / 网格切换。
- [ ] 提供批量选择入口。
- [ ] 提供添加内容入口。
- [ ] 剪贴板筛选激活时显示监听状态和暂停 / 恢复。
- [ ] 粘贴队列非空时显示数量入口。

### 6.3 列表视图

- [ ] 标准行高 84 pt，紧凑行高 64 pt。
- [ ] 行内展示类型、标题、两行预览、来源、时间、状态、标签、Pin、收藏。
- [ ] Hover 时显示快捷操作。
- [ ] 选中态使用极浅蓝背景。
- [ ] 支持键盘上下选择、Space 预览、Delete 删除、Return 主要操作。
- [ ] 图片和文件缩略图异步加载并缓存。

### 6.4 网格视图

- [ ] 卡片宽度 240-300 pt，高度约 188 pt。
- [ ] 图片显示缩略图。
- [ ] 链接显示站点图标、标题、摘要。
- [ ] 代码使用等宽字体和语言标签。
- [ ] 文件显示图标、名称和大小。
- [ ] 文本最多 5 行。
- [ ] 底部保留元信息区域。
- [ ] 1180 宽度下至少保留两列。

### 6.5 Inspector

- [ ] 无选中项时显示总数量、待整理数量、Pin 数量、手机同步状态、粘贴队列数量。
- [ ] 有选中项时显示完整内容、安全预览、来源、来源应用或设备、创建时间、类型、标签、项目。
- [ ] 提供 AI 操作：自动标题、摘要、提取待办、提取日程、润色、发送给 Agent。
- [ ] 提供去向：转任务、日程、知识库、Markdown、归档、删除。
- [ ] 底部固定主要操作区。
- [ ] 操作进行中显示具体进度和 Disabled 状态。
- [ ] 紧凑宽度下通过 Sheet 或 Overlay 展示。

### 6.6 Pin 与粘贴队列

- [ ] Pin 作为所有 `CollectedItem` 的通用能力。
- [ ] 支持 Pin 顺序调整。
- [ ] 支持显示全部、隐藏全部、关闭全部。
- [ ] 粘贴队列仅在非空时显示。
- [ ] 支持拖动或按钮调整顺序、单项移除、清空、连续粘贴。

### 6.7 批量操作

- [ ] 批量添加标签。
- [ ] 批量发送给 Agent。
- [ ] 批量转任务。
- [ ] 批量转日程。
- [ ] 批量归档。
- [ ] 批量导出。
- [ ] 批量删除必须二次确认。
- [ ] 部分失败时显示成功数、失败数和可重试项。

## 7. 状态、可访问性、性能

### 7.1 状态矩阵

- [ ] Shell、工作台、收集箱、Inspector 都有明确的 Loading、Empty、Error、Disabled、Permission 表达。
- [ ] Error 不用空白页替代。
- [ ] Disabled 必须解释原因。
- [ ] Permission Required 提供系统设置入口。

### 7.2 键盘与菜单

- [ ] `Cmd+F` 聚焦当前页面搜索。
- [ ] `Cmd+1...` 导航快捷键无冲突。
- [ ] Escape 退出批量选择或关闭预览。
- [ ] Delete 删除当前项并遵循危险操作确认。
- [ ] Sidebar、Toolbar、内容区和 Inspector 焦点顺序正确。

### 7.3 可访问性

- [ ] 图标按钮提供 accessibility label。
- [ ] 状态不仅依赖颜色表达。
- [ ] 支持 Increase Contrast。
- [ ] 系统字体放大后不截断关键操作。
- [ ] 装饰图片对辅助技术隐藏。
- [ ] 列表和网格项提供合并后的可读描述。

### 7.4 性能与稳定性

- [ ] 列表使用 Lazy 容器。
- [ ] 缩略图按需加载。
- [ ] 搜索和筛选不在 View body 中重复计算。
- [ ] ViewModel 任务在页面退出或查询变化时可取消。
- [ ] 系统状态轮询在工作台离开后停止或降频。
- [ ] 视图切换不重复启动剪贴板监听。
