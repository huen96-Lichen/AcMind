# AcWork Phase 1 详细推进任务单

> 目标：把 `615设计规范` 拆成可以逐项推进、逐项验收的执行单。  
> 范围：仅覆盖 Phase 1 的三块核心交付物
> - 统一应用外壳
> - 工作台
> - 合并后的收集箱
>
> 设计基准：1500 × 920 pt  
> 最小窗口：1180 × 720 pt  
> 响应阈值：1320 pt  
> 视觉风格：AcWork Focus Workspace

## 0. 执行前提

- [x] 保留现有用户改动，不回滚、不重置工作区。
- [x] 已完成基线记录、窗口尺寸基线和截图基线采集。
- [x] 已确认 Phase 1 只做三个核心交付物，不扩展到 Agent、日程、工具台等页面重制。
- [x] 已确认剪贴板不再作为一级页面，必须并入收集箱。
- [x] 已确认旧数据目录、Bundle ID、内部兼容标识暂不破坏性迁移。

## 1. 任务总顺序

1. 收口品牌、导航和窗口策略
2. 收口 Design Tokens
3. 建立统一基础组件
4. 完成统一应用外壳
5. 完成工作台
6. 完成收集箱统一模型
7. 完成收集箱 UI 重制
8. 完成状态、可访问性和交互收口
9. 完成截图、测试和验收

## P1 品牌、导航、窗口

### ACW-P1-01 品牌统一

**目标**

让用户可见文案统一为 AcWork，同时保留内部兼容标识。

**任务**

- [x] 应用标题、菜单、欢迎文案、关于页、设置页、通知展示文案统一为 `AcWork`。
- [x] 区分“用户可见名称”和“技术兼容标识”。
- [x] 保留 `AcMindKit`、Bundle ID、数据库目录、旧通知原值等兼容项。
- [x] 保留旧数据读取能力，不删除旧数据目录。
- [x] 新建组件和新代码逐步使用 `AcWork` 前缀。

**验收**

- 用户可见路径中不再出现旧品牌名。
- 兼容层不影响旧数据和旧通知。

### ACW-P1-02 一级导航重构

**目标**

把导航固定为 10 个一级页面，移除独立的“剪贴板”入口。

**任务**

- [x] 将 `.home` 映射为“工作台”。
- [x] 删除一级导航中的 `.clipboard`。
- [x] 保留旧 clipboard 路由兼容，但最终落到收集箱剪贴板筛选。
- [x] 统一四个导航分组：工作、处理、随身能力、系统。
- [x] 修复快捷键顺序和 `Cmd+2` 冲突。
- [x] 更新 App Intent、Companion、菜单和动态大陆的内部路由选择器。
- [x] 为旧持久化选中值增加 fallback，未知值回到工作台。

**验收**

- 侧边栏显示恰好 10 个一级页面。
- 不出现独立“剪贴板”导航项。
- 所有入口仍可达收集箱的剪贴板来源筛选。

### ACW-P1-03 窗口尺寸与恢复策略

**目标**

把主窗口行为固定到新基准，避免恢复旧窗口时越界或过小。

**任务**

- [x] 默认尺寸改为 1500 × 920。
- [x] 最小内容尺寸改为 1180 × 720。
- [x] 清理旧的 880 × 650 约束。
- [x] 校正历史窗口恢复时的小尺寸。
- [x] 保证屏幕可用区域不足时窗口仍可见。
- [ ] 为历史窗口恢复行为补一条覆盖测试。

**验收**

- 新启动默认尺寸正确。
- 旧窗口恢复不会出现裁切或屏幕外定位。

## P3 统一应用外壳

### ACW-P3-01 根布局收口

**目标**

让所有页面复用同一个稳定外壳，不在页面切换时重建结构。

**任务**

- [x] `ContentView` 只负责根布局和页面组合，不承载页面业务逻辑。
- [x] 固定 Sidebar 为 216 pt。
- [x] 固定 Toolbar 为 60 pt。
- [x] 页面体支持可选 Filter Rail、Main Content、Inspector。
- [x] 页面切换时不替换根布局。
- [x] 页面标题只在 Toolbar 出现一次。
- [x] 保留 Voice、Capture、Quick Note Sheet。
- [ ] 收口通知路由到命名清晰的导航方法。

**验收**

- 11 个页面共用同一个 Shell。
- Sidebar 和 Toolbar 切页时尺寸稳定。

### ACW-P3-02 Sidebar 视觉和行为

**目标**

把侧边栏收成稳定、可读、可键盘操作的导航壳。

**任务**

- [ ] 品牌区高度 72 pt。
- [ ] 导航项高度 38 pt。
- [ ] 导航项圆角 10 pt。
- [ ] 图标尺寸 16–18 pt。
- [ ] 分组标题 11 pt semibold。
- [ ] 选中态使用浅蓝背景 + 系统蓝图标。
- [ ] Hover 使用浅灰背景。
- [ ] 快捷键标签默认不常驻显示，仅在 Hover 时出现。
- [ ] 底部显示本地服务状态和当前模型。
- [ ] 支持键盘上下移动与激活。

**验收**

- 侧边栏无功能性跳动。
- 键盘路径可完整操作导航。

### ACW-P3-03 响应式与层级

**目标**

把三栏、双栏、单栏切换固定下来。

**任务**

- [ ] ≥1320 pt 允许三栏布局。
- [ ] 1180–1319 pt 隐藏 Inspector，详情改为 Sheet 或 Overlay。
- [ ] <1180 pt 禁止继续缩小。
- [ ] 高度 <760 pt 时收紧垂直间距，但不隐藏核心操作。
- [ ] 常规卡片不使用明显阴影，只保留 1 pt 描边。

**验收**

- 1180 × 720 下布局完整无裁切。
- 1320 pt 以上可进入三栏体验。

**验收**

## P5 收集箱统一模型

### ACW-P5-01 领域模型

**目标**

把多来源内容统一为一个展示和操作模型。

**任务**

- [ ] 定义 `CollectedItem`。
- [ ] 定义 `CollectionSource`。
- [ ] 定义 `CollectedContentType`。
- [ ] 定义 `ProcessingStatus`。
- [ ] 保证 `SourceItem` 与 `ClipboardItem` 的 ID 不冲突。
- [ ] 覆盖文本、链接、图片、文件、代码、富文本、视频。

**验收**

- 多来源内容能用同一套字段展示。

### ACW-P5-02 Adapter 层

**目标**

把旧模型映射到统一模型，而不是一次性重写底层存储。

**任务**

- [ ] `SourceItem` 映射到 `CollectedItem`。
- [ ] `ClipboardItem` 映射到 `CollectedItem`。
- [ ] 保留标题、预览、来源、时间、类型、标签等关键字段。
- [ ] 图片和文件只传递资源引用，不在列表层加载完整大文件。

**验收**

- 旧数据仍能显示在统一收集箱中。

### ACW-P5-03 Repository 聚合层

**目标**

把收集箱查询和操作统一到一个仓库接口。

**任务**

- [ ] 建立 `CollectedItemRepository`。
- [ ] 聚合 `StorageServiceProtocol` 与 `ClipboardServiceProtocol`。
- [ ] 支持列表、搜索、筛选、排序、分页或限制数量。
- [ ] 支持更新标签、Pin、收藏、归档、删除。
- [ ] 支持保存到知识库、发送给 Agent、转任务、转日程、导出 Markdown。
- [ ] 支持粘贴队列读取和操作。

**验收**

- UI 不再直接拼接两套存储的业务逻辑。

## P7 状态、可访问性、性能

**目标**

把所有页面的状态表达统一起来。

**任务**

- [ ] Shell、工作台、收集箱、Inspector 都有明确的 Loading、Empty、Error、Disabled、Permission 表达。
- [ ] Error 不用空白页替代。
- [ ] Disabled 必须解释原因。
- [ ] Permission Required 提供系统设置入口。

**验收**

- 用户能看懂为什么功能不可用。

### ACW-P7-02 键盘与菜单

**目标**

让核心操作可用键盘完成。

**任务**

- [ ] `Cmd+F` 聚焦当前页面搜索。
- [ ] `Cmd+1...` 导航快捷键无冲突。
- [ ] Escape 退出批量选择或关闭预览。
- [ ] Delete 删除当前项并遵循危险操作确认。
- [ ] Sidebar、Toolbar、内容区和 Inspector 焦点顺序正确。

**验收**

- 常用路径可以不依赖鼠标完成。

### ACW-P7-03 可访问性

**目标**

避免纯颜色依赖和缺失标签。

**任务**

- [ ] 图标按钮提供 accessibility label。
- [ ] 状态不只依赖颜色表达。
- [ ] 支持 Increase Contrast。
