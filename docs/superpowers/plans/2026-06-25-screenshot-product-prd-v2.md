# AcMind 截图功能 PRD v2

## Summary

本文档定义 AcMind 的截图产品化目标：把现有零散截图能力收束为一个完整、可持续扩展的截图功能，达到主流开源截图工具 90% 以上的核心功能和操作体验。

对标参考：
- [Flameshot](https://github.com/flameshot-org/flameshot)
- [ksnip](https://github.com/ksnip/ksnip)
- [Greenshot](https://github.com/greenshot/greenshot)
- [macshot](https://github.com/sw33tLie/macshot)

本文档明确以下问题：
- 设置和控制界面的入口在哪
- 截图快捷键默认是什么
- 怎么更改快捷键
- 截图存储路径在哪里
- 有几种截图方式
- 能不能调整截图预设

当前仓库版本入口以 `0.0.7` 为准，不沿用旧交接文档中的历史版本号。

---

## 1. Product Goal

AcMind 的截图功能目标不是“有截图按钮”，而是“用户可以像使用独立截图工具一样，快速完成捕获、预览、标注、隐私处理、保存、复制、Pin、回看”。

### 1.1 设计原则

- 快速触发：任何入口都应把用户带到同一截图流程。
- 低摩擦：截图后立刻进入预览，不强迫用户先学复杂面板。
- 本地优先：图片和元数据本地保存，不以云上传为前提。
- 可恢复：预设、快捷键、保存路径、历史都应可追踪。
- 可渐进增强：先补短闭环，再补标注、滚动、OCR、历史增强。

### 1.2 非目标

- 不在第一版实现完整图片编辑器。
- 不把云同步或云上传作为截图功能前提。
- 不把截图功能继续挂在工作台总览里作为“附属能力”。
- 不要求复制所有跨平台脚本和平台特化能力。

## 2. Current State

### 2.1 已存在的底层能力

- `CaptureService` 已支持全屏、区域、窗口、滚动截图，并在截图后执行后处理、打码、OCR、落库。
- `SettingsLocalPreferences` 已有截图开关、自动打码、打码模式、圆角、最大宽度、最大高度等字段。
- `AssetStore` 已作为统一的本地文件保存层，截图会落到本地 assets 目录。
- 设置页已有截图相关入口，但拆散在不同分类中，尚未形成独立的“截图产品面板”。

### 2.2 当前缺口

- 入口分散，用户需要自己理解“菜单、胶囊、设置页、热键”之间的关系。
- 截图预览层能力弱，缺少明确的操作工具条和编辑动作集合。
- `app.captureScreenshotHotkey` 仍是空默认值，不算完整产品入口。
- 已补命名预设集，并支持从空白模板新建预设。
- “窗口截图”在底层存在，但主入口未显式作为独立可见动作暴露。

## 3. Entry And Controls

截图功能必须有四条清晰入口线，每条线都要给出用户能看见的路径。

### 3.1 主设置页入口

路径：
- 设置
- 捕获与输入
- 截图捕获

作用：
- 控制截图总开关
- 控制自动打码
- 控制打码模式
- 控制圆角 / 最大宽度 / 最大高度
- 查看截图权限状态
- 查看截图保存行为的说明

### 3.2 随身能力入口

路径：
- 设置
- 随身能力
- 随身快捷键
- 截图捕获

作用：
- 录制和修改随身截图快捷键
- 作为面向高频用户的快速入口
- 与菜单栏、胶囊联动

### 3.3 菜单入口

路径：
- 菜单栏图标
- 快速采集
- 截图

作用：
- 最短路径唤起截图
- 适合无快捷键用户
- 适合临时操作

### 3.4 胶囊入口

路径：
- 桌面胶囊
- 截图
- 滚动截图

作用：
- 作为快捷控制面板
- 适合重复触发与高频操作
- 默认动作中应包含截图与滚动截图

### 3.5 设置页右侧预览区

设置页右侧预览区必须展示：
- 当前截图是否启用
- 当前截图快捷键
- 当前是否启用自动打码
- 当前保存路径是否有效
- 当前预设名称

目的：
- 用户不需要点进多层页面才能判断“截图是否真的可用”
- 避免只看到一个开关，却不知道操作链路是否生效

## 4. Hotkey Rules

截图功能需要区分两个不同的快捷键体系。

### 4.1 应用级截图热键

字段：
- `app.captureScreenshotHotkey`

默认值：
- 空

语义：
- 这是应用级全局热键
- 如果为空，则不注册
- 如果有值，则启动时自动注册

状态：
- 当前已经在“捕获与输入 -> 截图捕获”中提供可见编辑项
- 用户可直接输入或点击“录制”修改
- 保存后通过 `.settingsDidChange` 触发全局快捷键重新注册

### 4.2 随身截图快捷键

字段：
- `companionScreenshotShortcut`

默认值：
- `⌘⇧4`

语义：
- 这是随身能力里的截图快捷键
- 由设置页里的“随身快捷键”编辑
- 保存后立即生效

### 4.3 快捷键修改方式

#### 修改随身截图快捷键

路径：
- 设置
- 随身能力
- 随身快捷键
- 截图捕获
- 录制

行为：
- 打开快捷键录制弹窗
- 捕获用户按下的组合键
- 保存后立即生效
- 保存后刷新注册状态

#### 修改应用级截图热键

路径：
- 设置
- 捕获与输入
- 截图捕获
- 全局截图热键
- 输入或点击“录制”

行为：
- 空值表示不注册应用级截图热键
- 非空值保存到 `app.captureScreenshotHotkey`
- 保存后刷新全局快捷键注册状态

### 4.4 快捷键映射表

| 入口 | 字段 | 默认值 | 是否可录制 | 保存后是否立即生效 |
|---|---|---:|---|---|
| 应用级截图热键 | `app.captureScreenshotHotkey` | 空 | 是，已在截图捕获设置中可见 | 是 |
| 随身截图快捷键 | `companionScreenshotShortcut` | `⌘⇧4` | 是，已存在录制弹窗 | 是 |

## 5. Capture Modes

第一版必须明确暴露 4 种截图方式。

### 5.1 模式定义

| 模式 | 用户可见名称 | 底层状态 | 说明 |
|---|---|---|---|
| `fullscreen` | 全屏截图 | 已存在 | 直接截取主屏或当前屏幕 |
| `area` | 区域截图 | 已存在 | 用户拖拽选区后截图 |
| `window` | 窗口截图 | 已存在且主菜单已显式暴露 | 针对前台应用窗口截图 |
| `scrollScreenshot` | 滚动截图 | 已存在 | 通过滚动拼接生成长图 |

### 5.2 模式映射表

| 模式 | 触发入口 | 是否需要二次选择 | 是否进入预览 | 是否可编辑 | 最终保存方式 |
|---|---|---|---|---|---|
| 全屏截图 | 菜单、热键、胶囊、随身面板 | 否 | 是 | 待补标注编辑 | 预览后决定复制 / 保存 / Pin / 关闭 |
| 区域截图 | 菜单、胶囊 | 是，先选区 | 是 | 待补标注编辑 | 预览后决定复制 / 保存 / Pin / 关闭 |
| 窗口截图 | 菜单、胶囊 | 否 | 是 | 待补标注编辑 | 预览后决定复制 / 保存 / Pin / 关闭 |
| 滚动截图 | 菜单、胶囊、随身面板 | 可能需要页面滚动确认 | 是 | 待补标注编辑 | 预览后决定复制 / 保存 / Pin / 关闭 |

### 5.3 必须补齐的 UI

当前已补：
- 窗口截图已经在主菜单“快速采集 -> 截图 -> 窗口截图”中可见
- 滚动截图已经在主菜单和胶囊截图选项中可见
- 胶囊截图选项已经显示全屏、区域、窗口、滚动四类模式

仍待补：
- 预览层内的标注、箭头、文字、裁剪等图片编辑工具
- 更精细的窗口选择体验，而不是只截取前台窗口

## 6. Presets

当前实现已经从参数型预设升级为命名预设集。

### 6.1 已有可调参数

| 参数 | 字段 | 当前默认值 | 作用 |
|---|---|---:|---|
| 自动打码 | `captureAutoRedactionEnabled` | `true` | 控制是否对截图执行敏感内容处理 |
| 打码模式 | `captureCensorMode` | `pixelate` | 控制打码是像素化、模糊还是其他模式 |
| 圆角 | `captureScreenshotCornerRadius` | `0` | 控制输出图片圆角 |
| 最大宽度 | `captureScreenshotMaxWidth` | `0` | 控制输出最大宽度，`0` 表示不限制 |
| 最大高度 | `captureScreenshotMaxHeight` | `0` | 控制输出最大高度，`0` 表示不限制 |

### 6.2 当前“恢复默认”的真实行为

- 截图外观区的“恢复默认”只会把圆角、最大宽度、最大高度恢复为 `0`
- 预设区的“恢复默认预设”会恢复默认保存、复制优先、隐私优先三套命名预设
- 这两个入口必须保持文案区分，避免用户误以为外观恢复会重置整个预设集

### 6.3 已定义的命名预设

#### 1. 默认保存

- 自动打码：开
- 打码模式：像素化
- 圆角：0
- 最大宽度：原始尺寸
- 最大高度：原始尺寸
- 默认输出动作：保存到收集箱

#### 2. 复制优先

- 自动打码：关或按敏感级别开启
- 打码模式：模糊或像素化
- 圆角：0
- 最大宽度：原始尺寸
- 最大高度：原始尺寸
- 默认输出动作：复制到剪贴板

#### 3. 隐私优先

- 自动打码：开
- 打码模式：像素化优先
- 圆角：0 或较小圆角
- 最大宽度：可压缩
- 最大高度：可压缩
- 默认输出动作：保存到收集箱并保留历史

### 6.4 预设能力要求

- 预设可切换
- 预设可编辑
- 预设可恢复默认
- 预设变化应立即影响下一次截图
- 预设应与快捷键、入口解耦

### 6.5 预设 UI 规则

当前已实现：
- 当前预设名称显示
- 从空白模板新建预设
- 重命名预设
- 删除预设
- 恢复默认预设
- 复制预设
- 默认输出动作选择

## 7. Storage And Output

### 7.1 主存储路径

截图的主落盘目录是：

`~/Library/Application Support/AcMind/assets/`

该路径必须在设置页中可见，且文档里不得把临时兼容目录当作主目录宣传。

### 7.2 兼容路径

当前实现保留了临时目录候选路径用于兼容，不应作为主路径写入用户文档或设置说明中。

### 7.3 文件命名规则

| 类型 | 文件名规则 |
|---|---|
| 普通截图 | `screenshot_<timestamp>.png` |
| 滚动截图 | `scrollshot_<timestamp>.png` |

### 7.4 输出顺序

推荐顺序：
1. 采集截图
2. 保存到 assets
3. 打开预览层
4. 用户决定复制 / 保存 / Pin / 重截 / 编辑
5. 结果进入收集箱或历史

当前实现说明：
- 底层采集仍会先创建 `SourceItem` 和 asset，供预览读取
- 预览关闭或重截且用户未保存、未 Pin、未在 Finder 显示时，应清理这次临时 `SourceItem` 和 asset
- 用户点击保存、Pin 或在 Finder 中显示时，结果保留
- 标注编辑工具仍待补

### 7.5 输出动作

截图预览页至少要支持：
- 复制到剪贴板
- 保存到收集箱
- Pin 到桌面
- 重新截取
- 关闭

后续增强：
- 系统分享
- 导出为文件
- 发送到外部目标

## 8. UI And Workflow

### 8.1 截图产品结构

截图功能应重组为四层：

#### 入口层

- 菜单
- 热键
- 胶囊
- 设置页

#### 采集层

- 全屏
- 区域
- 窗口
- 滚动

#### 预览层

- 复制
- 保存
- Pin
- 重截

#### 处理层

- 打码
- 圆角
- 尺寸限制
- OCR

### 8.2 推荐用户流程

最短闭环：

1. 用户触发截图入口
2. 用户选择截图方式
3. 系统完成截图
4. 自动打开预览层
5. 用户选择复制 / 保存 / Pin / 编辑
6. 结果写入收集箱和历史

### 8.3 预览层要求

预览层不是“看图窗口”，而是截图操作中心。

必须包含：
- 当前图片预览
- 尺寸信息
- 保存状态
- 复制按钮
- 保存按钮
- Pin 按钮
- 重截按钮
- 关闭按钮

可选增强：
- 标注工具
- 打码工具
- OCR 结果面板
- 历史入口

## 14. Risks And Constraints

- `app.captureScreenshotHotkey` 当前是空默认值，文档不能假装它已经是完整用户入口。
