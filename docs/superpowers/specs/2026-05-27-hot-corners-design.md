# Hot Corners for Dynamic Continent

## Background

AcMind 曾经提供过屏幕四角触发能力。用户把鼠标移动到屏幕四个角之一并停留约 1.5 秒后，会触发每个角各自绑定的动作。这个能力目前在代码里只剩下“热区配置”的静态页面壳子，缺少真正的监听器、动作执行器和持久化数据。

This spec restores the feature and places its UI inside the existing “灵动大陆 & 配置” page.

## Goals

- 重新恢复四角触发能力，支持左上、右上、左下、右下四个角。
- 鼠标进入角区域后开始计时，停留满 1.5 秒触发动作。
- 每个角都可以绑定独立动作，不再只显示静态文案。
- 热区配置页放在“灵动大陆 & 配置”页面内。
- 配置可以持久化，并在应用启动后自动生效。
- 运行时逻辑和配置 UI 分离，便于后续继续扩展动作类型。

## Non-Goals

- 不在本次实现中做全量的脚本引擎或任意 shell 执行。
- 不把热区逻辑扩展到屏幕边缘的任意像素带。
- 不把它做成系统级全局快捷键替代品。

## UX

### Entry Point

- 在“灵动大陆 & 配置”页面中保留并升级“热区配置”section。
- 每个角显示一行摘要：
  - 角位名称
  - 当前绑定动作
  - 启用状态
  - 编辑入口

### Editing Flow

- 点击“编辑”后打开动作编辑器。
- 编辑器先选择动作类型，再根据动作类型展示对应参数。
- 推荐支持的动作类型：
  - 打开应用
  - 打开 URL
  - 切换内部功能
  - 打开应用内页面
  - 显示或隐藏某个面板
  - 无动作

### Feedback

- 角区域被悬停时，给出轻量高亮或状态提示。
- 满 1.5 秒触发时给出短反馈，例如 toast、顶部提示或状态回显。
- 动作失败时提示原因，不打断主窗口。

## Data Model

新增独立的热区配置模型，而不是把文案直接写死在页面里。

### HotCornerPosition

- `topLeft`
- `topRight`
- `bottomLeft`
- `bottomRight`

### HotCornerAction

采用可扩展的枚举 + payload 结构保存动作：

- `none`
- `openApp(bundleIdentifier)`
- `openURL(urlString)`
- `toggleFeature(featureIdentifier)`
- `openInternalRoute(routeIdentifier)`
- `showPanel(panelIdentifier)`

### HotCornerBinding

每个角保存：

- 位置
- 是否启用
- 悬停触发延迟，默认 1.5 秒
- 动作定义

### HotCornerSettings

保存四个角的完整配置，默认值如下：

- 左上角：无动作
- 右上角：展开灵动大陆
- 左下角：无动作
- 右下角：无动作

## Persistence

- 将热区设置作为应用设置的一部分持久化。
- 复用现有设置服务的读写路径，确保启动时自动加载。
- 持久化格式需要可版本化，便于后续新增动作类型。

建议将其作为 `AppSettings` 的一个字段，而不是散落在多个独立键里。这样可以：

- 一次性读写全部热区配置
- 避免角位和动作在升级过程中错位
- 方便后续做导入导出或恢复默认值

## Runtime Architecture

### 1. HotCornerManager

新增一个独立的运行时管理器，职责只有三个：

- 读取当前配置
- 监听鼠标位置变化
- 在满足悬停条件后执行对应动作

### 2. Hover Detection

- 监听系统鼠标移动事件。
- 计算鼠标当前是否进入四角的热区。
- 进入某个角后启动一次性计时。
- 在 1.5 秒内如果鼠标离开该角热区，则取消计时。
- 如果鼠标一直停留在同一个角，触发该角绑定的动作。

### 3. Trigger Guard

- 同一角触发后做短时间防抖，避免连续触发。
- 当配置被修改时，实时刷新运行时绑定。
- 当热区功能关闭时，立即停止监听。

### 4. Screen Awareness

- 以当前鼠标所在屏幕为准计算角区域。
- 支持多显示器环境。
- 当鼠标切换到其他屏幕时，热区监听自动切换到新屏幕。

## Integration Points

### DynamicContinentConfigView

- 将现有“热区配置”section改成可编辑 UI。
- 每个角的绑定通过该页面修改并保存。

### App Startup

- 应用启动后，设置服务加载热区配置。
- 若热区功能启用，则启动 `HotCornerManager`。

### Action Execution

- 打开应用动作通过系统 API 执行。
- 打开内部页面或面板通过现有路由/窗口管理入口执行。
- 切换功能动作通过现有服务层完成，不直接让 UI 互相引用。

## Error Handling

- 应用不存在时提示“未找到目标应用”。
- URL 无效时提示“链接无效”。
- 内部功能不可用时提示对应模块未启用。
- 执行动作失败时只反馈当前角，不影响其他角的配置和监听。

## Testing

### Unit Tests

- 四角命中测试：鼠标位置是否落入正确角位。
- 悬停计时测试：进入后满 1.5 秒触发，离开后取消。
- 动作序列化测试：读写后动作类型和参数保持一致。
- 默认值测试：四个角的默认绑定符合预期。

### Integration Tests

- 启动后配置自动加载。
- 修改热区配置后，运行时立即更新。
- 多屏场景下，角位识别基于当前屏幕。

## Rollout Plan

1. 先补数据模型和持久化。
2. 再实现 `HotCornerManager` 的监听和触发。
3. 然后把 “灵动大陆 & 配置” 页面里的热区 section 改成可编辑。
4. 最后补测试和基本反馈提示。

## Acceptance Criteria

- 四个角都能绑定独立动作。
- 鼠标停留 1.5 秒后会触发动作。
- 配置能保存并在重启后恢复。
- 热区配置页出现在“灵动大陆 & 配置”页面中。
- 启用/禁用热区后，监听行为能正确开关。
