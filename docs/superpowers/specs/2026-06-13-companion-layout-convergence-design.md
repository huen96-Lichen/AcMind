# 灵动大陆布局收敛设计

## 目标

通过删除冗余区块，减少灵动大陆各页面的垂直空间浪费，并让每个页面都只围绕一个主线展开。这是一次纯收敛调整，不引入新的入口去向，也不会在本次迭代里重新安置被删掉的内容。

## 问题

当前的概览页、音乐页、AI 页和状态页都放了多个小型辅助卡片，导致标题重复、信息重复，主内容下面还会出现明显空白。结果就是页面顶部很满，整体却不够紧凑。

## 范围

### 概览页

- 移除 `快捷入口` 卡片。
- 保留当前任务卡作为左侧主焦点。
- 保留运行摘要卡和系统总览卡。
- 保留现有顶栏导航和状态胶囊。

### AI 页

- 移除 `AI 状态` 卡片。
- 移除 `快捷指令` 卡片。
- 将 `对话历史` 移到左侧，让历史成为主要的辅助内容。
- 保留中间的对话卡作为页面主焦点。
- 保留对话卡内部现有的快速输入区域。

### 音乐页

- 移除 `来源与队列` 卡片。
- 移除 `歌词` 卡片。
- 保留 `正在播放` 作为主卡片。
- 保留 `播放控制` 作为唯一的辅助卡片。
- 保留播放控制、封面、标题、进度和前后切歌按钮。

### 状态页

- 移除左侧的 `关键指标` 卡片。
- 将 `运行摘要` 移到左侧，让它成为主要摘要块。
- 保留右侧的 `本机状态`。
- 保留进入完整状态窗口的跳转操作。

## 非目标

- 不重新设计顶栏入口。
- 不为被移除的页面内容新增新的去向。
- 不修改正在播放检测、播放状态获取或媒体适配器接入逻辑。
- 本次不改收缩态灵动大陆布局。

## 设计原则

- 每个页面尽量只有一个主内容块。
- 同一页面避免重复摘要。
- 让页面高度被内容密度消耗，而不是装饰性空白。
- 保持现有深色玻璃风格的一致性。

## 验证

- 布局修改后重新构建应用。
- 确认每个页面都还能正确渲染预期的主卡片。
- 检查音乐页在收缩态和展开态都还能显示当前播放信息。
- 确认状态页的跳转仍然可以打开完整状态窗口。
