# UI Shell 变更前后对照

## 目标
- 说明本轮“结构梳理清理”具体改变了什么。
- 帮助评审快速识别稳定性收益与后续约束。
- 当前对照只描述活跃实现，历史迁移残留路径不再作为新实现基线。

## 壳层结构
- 之前：
  - `Workspace / Secondary / Settings / AppShell` 存在各自容器实现，滚动区、padding、对齐策略不一致。
  - 同宽度下页面行为不一致，切页时更容易出现布局跳变。
- 现在：
  - 统一为 `header + body` 骨架。
  - 统一滚动容器参数来源（`pagePaddingX/Y/Bottom` + `maxWidth/alignment/spacing` 可配置）。
  - 页面差异收敛为参数差异，而不是骨架分叉。

## 一级菜单（Primary Rail）
- 之前：
  - 紧凑/展开模式存在结构和尺寸策略差异。
  - 中段/底部高度分配依赖 `Spacer` 等补偿，窗口变化时有裁切风险。
- 现在：
  - 两模式共用同一壳体结构（top/middle/bottom）。
  - 中段为可滚动主弹性区，底部为固定节奏区。
  - 去掉造成高度抖动的分配方式，降低裁切概率。

## 一级菜单项几何与动效
- 之前：
  - hover/selected 触发几何变化（缩放等），视觉基线容易跳。
  - 紧凑/展开的图标占位与内边距不完全一致。
- 现在：
  - 行高固定：`ACLayout.primaryRailNavItemHeight`。
  - 胶囊背景高度固定：`32`。
  - 图标占位固定：`20x20`。
  - 状态变化只影响颜色/阴影，不再改变几何尺寸。

## 图标与文本基线
- 之前：
  - 图标渲染模式可能受 SF Symbol 默认层级影响，重心不统一。
- 现在：
  - 一级菜单图标统一 `symbolRenderingMode(.monochrome)`。
  - 文本基线偏移统一策略，图标与文案中心线更稳定。

## 稳定性收益
- 裁切类问题触发条件显著减少（尤其在小高度、模式切换、页面切换场景）。
- 页面壳层行为更可预测，后续新增页面更容易复用规范。
- 视觉抖动降低，一级菜单交互一致性提升。

## 后续接入约束（给 CR 用）
- 禁止新增页面自定义壳层分叉，优先复用统一骨架。
- 禁止一级菜单项用 `scale/offset/height` 做 hover/selected 几何动画。
- 新增一级菜单项必须遵循固定行高/图标占位/胶囊高度规则。
- 壳层和菜单改动必须附带一次 `xcodebuild` 结果与窗口切换手测记录。
